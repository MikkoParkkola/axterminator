## Three-tier VLM architecture (locked from S1)

The operator's framing crystallizes here. Different latency budgets + different output classes need different model classes:

| Tier | Purpose | Latency budget | Param count | Memory | Owner |
|---|---|---|---|---|---|
| **Tier 1 — Fast UI grounding** | axterminator hot path: "find Submit button", "read error text", "is spinner visible" | < 200 ms p95 | 0.5-2B | 200-500 MB VRAM | **MIK-4379 (this ticket)** |
| **Tier 2 — Vision verification** | metacognition rubric scoring on UI screenshots | 200-400 ms | 2B | 1-2 GB VRAM | MIK-4376 (Cosmos Reason2) |
| **Tier 3 — Video understanding + generation** | botnaut native multimodal substrate | 500 ms / frame | 2B + 2B | 4-8 GB VRAM | MIK-4377 (Cosmos Reason2 + Predict 2.5) |

🟢V — every tier cites its owning ticket. The three tiers are **non-overlapping** by latency × workload class. Each operates inside its own resource envelope without contending with the others.

## PICK.1 — provisional primary

**Provisional pick: Moondream2** (1.9B full, ~0.5B quant).

Rationale:
1. **Apache 2.0** confirmed (no license risk).
2. **Designed-for-UI-screenshots + OCR + point-and-click grounding** per ticket — the only candidate with this explicit design focus.
3. **Quant path to ~0.5B** keeps the model within the Tier-1 memory envelope (200-500 MB VRAM target).
4. **HuggingFace Transformers integration path** is straightforward (per S2 model card).

**Conditional**: this pick is provisional until AUDIT.1 numbers land. Tie-breaker rule already codified per S1: "Default tie-breaker: Moondream2 (designed-for-UI fit)." If AUDIT.1 surfaces a measured gap > 10% on UI-grounding accuracy vs SmolVLM, revisit.

## INTEGRATE.1 — MCP surface (design only)

axterminator MCP server gains three new tools:

```rust
fn describe_screen(screenshot: ImageBytes) -> String;           // full caption
fn find_element(screenshot: ImageBytes, query: String) -> Option<UiElement>;  // bbox + text
fn verify_ui_state(screenshot: ImageBytes, expected: String) -> StateMatch;   // boolean + confidence
```

### Coexistence with Accessibility API (locked)

**Accessibility API remains primary path.** VLM is invoked ONLY when:
- Accessibility API returns empty for a known-populated screen (web app, Electron, browser content rendered in a non-AX-friendly way)
- Accessibility API returns ambiguous (multiple identical-name buttons; visual context disambiguates)
- Operator explicitly invokes `find_element` with a visual cue ("the red Save button")

This preserves the existing zero-overhead AX path for 80%+ of axterminator queries and adds VLM as a strict-fallback. No GPU contention with the rest of the agent loop because VLM is OFF until invoked.

## LATENCY.1 — target ratification

< 300ms p95 end-to-end on operator macOS hardware (Apple Silicon M-series).

Breakdown:
- Screenshot capture (existing AX path): ~10 ms
- VLM inference: ~150-250 ms (Moondream2 quant on Apple Silicon; 🟡I pending SPIKE)
- Structured-output formatting + JSON serialization: ~10 ms

Headroom: ~30-130 ms. SmolVLM-2.2B may not fit the budget at quant; Moondream2 quant has stronger latency story per S2 (designed for sub-200ms inference).

## DEVIL + STEELMAN

**DEVIL**: yet another VLM in the portfolio. Three different VLMs (Tier-1 small + Tier-2 Cosmos Reason2 + Tier-3 Cosmos Predict) is three more models to maintain, three sets of weights to download, three sets of failure modes.

**STEELMAN**: the latency × workload-class envelope is non-overlapping. Tier-1 cannot use Cosmos (too heavy); Tier-2/3 cannot use Moondream2 (insufficient reasoning depth). Sharing a single VLM across all tiers would either degrade Tier-1 latency or starve Tier-2/3 of reasoning quality. The 3-tier architecture is the cost of doing multimodal right.

## Risk + rollback

- **Risk**: Moondream2 quant fails to hit 200ms on M-series → fallback to SmolVLM-2.2B
- **Risk**: AX-fallback heuristic mis-fires (sends easy AX query to VLM unnecessarily) → instrument fallback-trigger telemetry; tune
- **Risk**: VRAM footprint exceeds 500 MB during agent-loop coexistence → enforce lazy-load + unload on idle
- **Rollback**: feature flag `AXTERM_VLM_ENABLED=0` reverts to AX-only behavior
